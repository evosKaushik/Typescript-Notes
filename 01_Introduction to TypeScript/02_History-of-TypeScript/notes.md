# 📘 History of TypeScript

## 🚀 Why TypeScript Was Born

JavaScript became much faster thanks to modern engines:

* Google → **V8**
* Microsoft → **Chakra**
* Mozilla → **SpiderMonkey**

These engines made JavaScript **10x–100x faster**, making large web applications possible.

Microsoft wanted complex applications like:

* Word
* Excel
* PowerPoint

to run directly in the browser.

However, JavaScript had challenges for large projects:

* Spelling mistakes
* Missing properties
* Runtime errors
* Difficult maintenance at scale

---

## 🤔 New Language or Improve Existing?

Some companies chose replacement languages.

Example:

* Google → **Dart**

Microsoft chose a different path:

👉 Improve JavaScript's developer experience instead of replacing JavaScript.

---

## 👨‍💻 Anders Hejlsberg & The Team

TypeScript started as an experiment by:

* Steve Lucco
* Luke Hoban

Later, **Anders Hejlsberg** (creator of C#) joined and guided the language design.

His vision:

> "TypeScript should not replace JavaScript."

---

## 🎯 Core Design Decisions

TypeScript was designed as:

* ✅ A strict superset of JavaScript
* ✅ Optional static typing
* ✅ Compiles into plain JavaScript
* ✅ Runs in every browser

---

## 🖥️ VS Code & TypeScript

TypeScript and VS Code grew together.

VS Code constantly stress-tested TypeScript and provided valuable feedback that helped improve the language.

---

## 🌍 Open Source Challenges

In 2012, Microsoft was not known for being open-source friendly.

Many developers asked:

> "Is Microsoft trying to embrace, extend, and extinguish JavaScript?"

Even inside Microsoft, convincing people was difficult.

---

## 🎉 Official Launch

📅 **October 1, 2012**

TypeScript was officially announced at the **GOTO Conference (Denmark)**.

Initially, it was hosted on **CodePlex**, Microsoft's open-source platform.

The launch was small.

No huge hype.

No instant takeover.

But something important had started.

---

## 🔄 CodePlex → GitHub

TypeScript originally lived on CodePlex.

In **2014**, the project moved to **GitHub**, helping it gain wider community adoption.

---

## 📦 DefinitelyTyped

**DefinitelyTyped** is a repository containing type definitions for JavaScript libraries.

Without it:

* TypeScript would be isolated.

With it:

* Existing JavaScript libraries could be typed and used safely.

Later, npm's **@types** packages made installation seamless.

---

## ⚔️ Rival Philosophies

Different companies had different ideas:

* Google → Dart (replace JavaScript)
* Facebook → Flow (type checking)
* Microsoft → TypeScript (enhance JavaScript)

---

## 🚀 Angular & The Adoption Explosion

A major turning point came when **Angular 2** adopted TypeScript.

This created collaboration between:

* Microsoft
* Google
* Angular Team

TypeScript adoption grew rapidly.

---

## 🌱 Ecosystem Strategy

TypeScript worked with existing tools instead of competing with them:

* Angular
* React
* Vue
* Svelte
* Babel
* esbuild
* Deno

---

## 🔥 Self-Hosted

TypeScript is written in **TypeScript itself**.

This is called **self-hosting**.

---

# 📝 Final Notes

✅ TypeScript was **not created to replace JavaScript**.

✅ JavaScript was already powerful enough for complex applications.

✅ TypeScript was created to improve:

* Developer Experience (DX)
* Productivity
* Maintainability
* Tooling
* Large-scale application development

### One-Line Summary

> TypeScript succeeded because it improved JavaScript instead of fighting against it.
